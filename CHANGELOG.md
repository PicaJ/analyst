# analyst 更新日志

## v0.8.1 — 2026-05-11

### 输出稳定性优化: 消除 LLM 温度导致的洞察数量波动

**问题**: 同样 30 天数据，每次运行输出洞察数从 17~41 条大幅波动 (24 点差距)
- 根因: LLM temperature=0.3 导致同一链每次生成不同 thesis 措辞
- 波动路径: 52 chains(稳定) → 去重(波动主因) → 过滤(级联放大)

**第 1 层: Pre-LLM 链合并 (agent.py `_merge_overlapping_chains`)**
- 在 LLM 分析之前，合并主题重叠的链 (确定性操作)
- Union-Find 分组: Jaccard ≥ 0.25 node overlap 的链合并为一条
- 效果: 51~53 chains → 43~45 merged chains (每次运行一致)

**第 2 层: 结构化去重 (agent.py `_deduplicate_insights`)**
- 新增 chain_id 分组去重: 同一链只保留最高 confidence
- 新增 chain_theme 门控: 不同主题的洞察不做文字去重
- 提高 LCS 阈值: 10 → 25 (只匹配真正相似的论点)
- 提高 Jaccard 阈值: 0.35 → 0.50 (3-gram)

**第 3 层: 过滤稳定化 (agent.py `_filter_low_quality_insights`)**
- 新增 significance 锚定: chain_significance ≥ 0.5 时 confidence 阈值从 0.3 降到 0.2

**验证结果 (3 次连续运行，相同数据)**
| 阶段 | Run 3 | Run 4 | Run 5 |
|------|-------|-------|-------|
| Merge | 51→43 | 51→43 | 51→43 |
| Chain-id dedup | 43→40 | 43→40 | 43→40 |
| Thesis dedup | 40→22 | 40→24 | 40→19 |
| Filter | 22→16 | 24→16 | 19→14 |
| **Final** | **16** | **16** | **14** |

- 洞察数: **14~16 条** (2 点差距, 目标 ≤3)
- 对比优化前: 17~41 条 (24 点差距)
- Pre-LLM 阶段完全稳定 (merge=43, chain-id=40)
