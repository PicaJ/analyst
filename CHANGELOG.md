# analyst 更新日志

## v0.6.0 — 2026-05-10

### 架构级改进: 覆盖率提升

**问题诊断**
- 30天非公告新闻 16,922 条，仅 333 条进入链 (覆盖率 2.0%)
- 82.8% 的新闻不包含任何 tracking_keyword，完全未被覆盖
- 非公告新闻结构化字段全部为空 (`mentioned_companies=[]`, `related_sectors=[]`)
- hot_keyword_threshold=80 过高，单个热词难以达到

**改进1: 实体抽取富化 (chain_builder.py)**
- 新增 `enrich_items()` 方法: 从标题中提取公司名(→ts_codes) + 推断行业板块
- 使用编译正则匹配 ~5000 个公司名（>=3字符，避免泛词误匹配）
- 在所有链构建方法中调用 (build_timeline/sector/anomaly/entity_cross)
- 配合已有 `NewsEnricher` 的 Tier1 规则富化

**改进2: 语义主题发现链 (chain_builder.py)**
- 新增 `build_semantic_theme_chains()` 方法
- 利用已有 FAISS 向量索引对非公告新闻做 K-Means 聚类
- 自动发现 tracking_keywords 之外的新兴投资主题
- 过滤已覆盖簇(>30% tracking_keyword命中) 和停用词主导簇
- 新链类型: `semantic_cluster`，上限可配置 (`max_semantic_chains`)

**改进3: 公司级链构建 (agent.py)**
- 从公告 ts_codes 统计高频公司（>=5条公告），自动建公司 timeline 链
- 例如：601006.SH(35条公告)、688625.SH(34条公告) 等
- 上限可配置 (`max_company_chains`)

**改进4: 降低高频词阈值 (analyst.yaml)**
- `hot_keyword_threshold`: 80 → 20
- 使更多市场热词有机会被建链

**配置新增 (config.py / analyst.yaml)**
- `max_semantic_chains: 2` — 语义主题链上限
- `max_company_chains: 2` — 公司级链上限
- `max_timeline_chains: 18` — timeline 链上限(含公司链)

## v0.5.1 — 2026-05-10

- 概念股行情: `概念股拉升`、`概念股活跃`、`概念股大涨`、`概念股涨停`、`概念股走强`
- 趋势描述: `震荡上扬`、`震荡下行`、`震荡走高/低`、`持续上行/下行`、`股价创/突破`、`板块回调/分化`
- 资金流: `涨停板`、`跌停板`、`封板`、`开板`、`主力净流入/流出`、`龙虎榜`、`ETF涨幅/跌幅`

**chain_stop_words / _STOP_WORDS 新增 20+ 财务泛词**

- 财报报告: `财报`、`一季报`、`年报`、`半年报`、`季报`、`业绩快报`、`业绩预告`、`财报季`、`披露期`
- 业绩泛词: `业绩`、`业绩公告`、`业绩报告`、`季度报告`
- 季度代号: `Q1`、`Q2`、`Q3`、`Q4`
- 增长泛词: `同比增长`、`环比增长`、`盈利`、`亏损`、`扭亏`、`派息`、`分红`
- **效果**: 彻底消除"净利润事件时间线"(40分→消失)等财务指标链，此类关键词匹配所有公司财报，无法形成有效投资因果链

### 澄清公告过滤扩展 (chain_builder.py)

**_CLARIFICATION_KEYWORDS 新增 7 个否定模式**
- `未生产`、`未销售`、`不生产`、`不销售`、`未用于`、`未供应`、`无业务`
- **效果**: 过滤"东利机械未生产销售用于人形机器人的减速器"、"中密控股明确表示产品未用于火箭"等零投资价值公告

### 商品比喻/噪声过滤增强 (chain_builder.py)

**矿 metaphor rules 扩展**
- never_if 新增: `印度`、`越南`、`澳大利亚`、`莫迪`、`外交`、`访问`、`合作开发`
- never_if 新增: `挖矿`、`比特币`、`加密货币`
- must_have 新增: `有色金属`
- **效果**: 过滤"印度越南合作矿产开发"等外交新闻，过滤"加密货币挖矿"等无关新闻

### 剩余已知限制

以下低分链由关键词本质特性决定，非代码过滤可解决：
- **海外关键词 (GPT/苹果/Anthropic/谷歌)**: 天然匹配大量非A股国际新闻，chain_score 偏低 (35-68)
- **低数据量关键词 (白银)**: 可匹配新闻过少，链信号不足 (35-58)
- **宽泛主题 (AI/AI应用)**: 覆盖面太广，导致信息密度稀释
- **LLM 评分波动**: 同一代码、同一数据，10轮 chain_score 均值 70.9，极差 7.1 分 (67.3~74.4)

## v0.5.0 — 2026-05-10

### 链构建质量大幅优化 (chain_builder.py)

**行情播报噪声过滤增强**
- 扩展 `_is_quote_broadcast()` 为实例方法，新增 `_BROADCAST_TITLE_PHRASES` 二级行情短语表
- 新增 30+ 期货行情过滤词：`转跌`、`转涨`、`跌幅收窄`、`WTI`、`布伦特`、`ETF持仓`、`交易量统计` 等
- 新增 ETF 行情过滤：`ETF涨`、`ETF跌`、`ETF资金`、`基金发行`、`基金规模`
- 新增短线行情过滤：`冲高回落`、`探底回升`、`窄幅震荡`、`主力资金`、`北向资金`、`半日主力` 等
- 新增盘中简讯过滤：`盘中异动`、`券商晨报`、`券商研报`、`晨会纪要`
- 过滤层级从 4 层扩展到 7 层，新增 Layer 5 关键词相关性、Layer 6 同事件去重、Layer 7 短时间窗口碎片去重

**澄清公告过滤**
- 新增 `_CLARIFICATION_KEYWORDS`：`澄清`、`不涉及`、`无关联`、`未开展`、`未有`、`与本公司无关`、`未与`
- 在 `_filter_for_chain()` Layer 3 中排除此类零投资价值的公告

**同事件多源去重**
- `_dedup_same_event()` 标题相似度 >80% 视为同一事件，保留最高优先级源

**短时间窗口碎片去重**
- 新增 `_dedup_time_clustered()`：同一 source 在 30 分钟内 >3 条新闻只保留 priority 最佳的
- 解决"美联储官员讲话半小时拆成 24 条碎片"的问题

**商品比喻用法过滤**
- 新增 `_COMMODITY_METAPHOR_RULES` 字典，覆盖 `黄金`、`白银`、`火箭`、`矿` 四个易混淆关键词
- `黄金`：排除"黄金通道/黄金五月"等形容词用法，要求同时出现"金价/金矿/黄金ETF"等资产相关词
- `白银`：同理排除"白银时代"等比喻
- `火箭`：排除"火箭弹/火箭炮/导弹"等军事新闻，要求"商业航天/可回收/发射"等航天相关词
- `矿`：排除"钙钛矿/煤矿安全"等非投资相关新闻，要求"锂矿/稀土/碳酸锂"等矿业相关词
- 在 `_filter_by_keyword_relevance()` 中增加 Layer 0 商品比喻过滤

**实体交叉链噪声根治**
- `build_entity_cross_chains()` 不再使用 `_extract_title_keywords()` 从标题提取碎片关键词
- 仅使用结构化字段 `mentioned_companies` 和 `related_sectors` 作为实体来源
- 重叠阈值从 3 提高到 4
- 相关实体提取同样仅使用结构化字段
- 彻底消除 "AI→中信" (25分)、"AI→商业" (55分) 等垃圾交叉链

**异常链碎片过滤**
- `_detect_entity_bursts()` 中标题关键词最小长度从 2 提高到 3 字符
- 消除 "高质量/青年/引擎/致贺/时代" 等两字碎片的异常链

**停用词表大幅扩充**
- 新增 100+ jieba 分词碎片停用词：投资/研报/高端/商业/布局/核心/持续/加速...
- 新增券商简称停用词：中信/建投/国泰/海通/华泰/招商/广发/申万/银河/中金
- 新增公司名碎片：伟达/英伟/特斯/亚马...
- 新增泛词碎片：高质量/青年/引擎/致贺/时代/板块/概念/赛道/风口...

### LLM 荐股准确性修复 (insight_engine.py)

**股票代码验证三层机制**
- 新增 `_stock_info_cache` 缓存，从 `stock_industry_cache.json` 和 `ts_code_name.json` 加载 code→{name,industry} 映射
- Layer 1 (虚构检测)：LLM 推荐的 ts_code 不在链中实际出现的 `available_ts_codes` → 标记 `fabricated=True`
- Layer 2 (行业匹配)：推荐股票的实际行业不在 LLM 的投资论点中 → 标记 `business_mismatch=True`
- `_verify_recommendations()` 方法在 LLM 输出后自动执行，移除不合格推荐

**Prompt 工程强化**
- SYSTEM_PROMPT 中 "优先从...选取" 改为 "必须且只能从...列表中选取，禁止使用列表之外的股票代码"
- 新增 "股票信息准确性要求" 章节，要求 LLM 必须使用提供的真实公司名称和行业
- CHAIN_ANALYSIS_PROMPT 中股票代码展示格式改为 `000333.SZ → 美的集团（白色家电）`
- 移除 "2-5只" 上限限制，改为 "按推荐强度从强到弱排列"

### 配置与报告 (config.py / analyst.yaml / report.py)

- 新增 `llm_per_chain_timeout: 300` 配置项，单条链 LLM 分析总耗时安全阀
